OpenGL学习笔记（八）：HDR转SDR

发表于2023-08-02|更新于2024-02-24
|阅读量:

简介

显示设备是否支持HDR取决于其硬件和软件能力。许多旧的显示设备和一些低端设备可能不支持直接的HDR显示

HDR（高动态范围）图像通常使用更高的比特深度来表示亮度和色彩信息，以提供更丰富的细节和更广泛的亮度范围。这是一种拥有比普通RGB格式图像（仅8bit的亮度范围），常见的HDR图像格式包括10位、12位和16位等。

SDR（标准动态范围）图像通常使用较低的比特深度来表示亮度和色彩信息。常见的SDR图像格式包括8位和10位等。

PS:HDR（高动态范围）的10位可以提供比SDR（标准动态范围）的10位更多的亮度和色彩值，主要是因为HDR使用了更广泛的亮度范围和更丰富的色彩空间。具体来说，HDR使用了更高的亮度范围，通常可以达到1000到10000尼特的亮度，而SDR的亮度范围通常只有100到1000尼特。这意味着HDR可以显示更多的亮度级别，从而提供更多的亮度细节和更好的对比度。此外，HDR还使用了更丰富的色彩空间，如Rec.2020和DCI-P3等，相对于SDR的BT.709色彩空间，这些色彩空间可以提供更多的色彩细节和更广泛的色彩范围。因此，HDR的10位可以表示更多的色彩值，从而提供更丰富的色彩细节和更真实的色彩表现。尽管HDR和SDR的10位具有相同的比特深度，但由于使用不同的色彩空间和亮度范围，HDR在每个RGB通道的取值范围会大于SDR.

有一些API可以用于获取显示屏是否支持HDR：

  • Windows API：在Windows操作系统上，可以使用Windows Display API来获取显示屏的HDR支持情况。具体来说，可以使用GetDisplayConfigBufferSizes函数和QueryDisplayConfig函数来获取显示屏的配置信息，包括HDR支持情况。
  • macOS API：在macOS上，可以使用Core Display API来获取显示屏的HDR支持情况。具体来说，可以使用CGDisplayCopyDisplayMode函数和CGDisplayModeCopyPixelEncoding函数来获取显示屏的像素编码和HDR支持信息。
  • Android API：在Android操作系统上，可以使用MediaCodecInfo类和MediaCodecList类来获取显示屏的HDR支持情况。具体来说，可以使用MediaCodecList类的getCodecInfos函数来获取所有编解码器的信息，然后使用MediaCodecInfo类的getCapabilitiesForType函数来获取特定编解码器的能力信息，包括HDR支持情况。
